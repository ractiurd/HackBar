## Prologue: The Legacy

If you have been in the web security space for more than a few years, you know the name. HackBar — the quintessential Firefox extension that defined how penetration testers, bug bounty hunters, and security researchers interacted with web applications.

Originally developed by **PhHitachi**, the HackBar Firefox extension was an indispensable part of every pentester's toolkit. It provided a clean, intuitive interface for crafting and executing payloads, encoding and decoding data on the fly, and testing for common web vulnerabilities.

Over time, as browsers evolved, the original extension remained tied to older Firefox builds and forks like Cyberfox. Many pentesters kept an antique browser installation solely for the sake of using HackBar. The workflow became fragmented, and the community longed for a modern equivalent.

## 👻 GhoulSight (External XSS Scanner)



**GhoulSight** is a high-performance XSS vulnerability scanner that combines traditional scanning with advanced payload generation to deliver **verified results with zero false positives**.

> **⚠️ Important:** GhoulSight is a **separate external tool** that must be be installed and run alongside the HackBar extension. It is **not bundled** with HackBar.

> **🚧 Current Status:** GhoulSight is currently available as a **terminal (CLI) application only**. Browser integration is under active development and will be introduced in a future update.

- **Repository:** [github.com/ractiurd/ghoulsight](https://github.com/ractiurd/ghoulsight)
- **Language:** Go 1.25
- **License:** Subscription‑based (trial mode available)

### Key Features

- **Verified Results** – Every XSS finding is validated through real execution, eliminating false positives
- **Smart Scan** – Stops testing a URL after the first confirmed XSS is found
- **Multi‑Mode Scanning** – Reflected (`r`), DOM (`d`), Context‑Aware (`c`), and Fuzz (`f`) modes
- **Context‑Aware Engine** – AI‑powered context detection with mutation engine
- **Domain Crawler** – Automatically discovers URLs with parameters
- **Path‑Based Detection** – Injects payloads into URL path segments
- **Telegram Integration** – Real‑time scan notifications

### Setup in HackBar

1. **Install GhoulSight** – Clone and build the tool from its repository
2. **Enter your API Key** and **Server URL** in the GhoulSight settings panel (click the `☰` button)
3. **Toggle the scanner** (`Alt+G`) to start scanning the current page
4. **Live Stats** – The summary strip shows `sent` (requests), `jobs` (submitted to server), `dedup` (duplicate signatures skipped), and `POC` (confirmed XSS)
5. **Results** – Confirmed XSS opens a dedicated popup window with the PoC and an "Execute in Tab" button

> **Note:** GhoulSight does **not** come with built‑in default payloads – you must provide your own payload files. The Context‑Aware mode (`-m c`) generates payloads dynamically and does not require a payload file.

## Installation

As this is a developer‑focused tool, installation is performed via the browser's developer mode:

1. Clone this repository or download the ZIP archive.
2. Open your Chromium‑based browser and navigate to the extensions page:
   - `chrome://extensions/` (Chrome, Brave, Opera)
   - `edge://extensions/` (Edge)
3. Enable **Developer mode** (toggle in the top‑right corner).
4. Click **Load unpacked** and select the folder containing the `manifest.json` file.
5. The HackBar icon will appear in the toolbar. Click it to open the side panel.

## Advanced Tools Deep‑Dive

### 🔍 GAP (Recon)
Ported directly from the Burp Suite extension:
- **Links**: Extracts absolute, relative, and JS‑built URLs. Click to copy, double‑click to open.
- **Params**: Discovers parameters from URLs, forms, JSON, XML, HTML inputs, and JS variables. Susceptible parameters are marked with vulnerability types (e.g., `XSS`, `SQLi`, `SSRF`).
- **Words**: Generates a unique wordlist from the page content. Supports plurals, lowercase variants, path/parameter inclusion, and custom stop‑words.
- **Layout**: Adjustable 3‑panel grid (Params | Words on top, Links full‑width below) with drag‑to‑resize handles.

### 🔧 Admin Fuzzer
- **FUZZ Token**: Place `FUZZ` anywhere in the URL (e.g., `http://example.com/FUZZ` or `?id=FUZZ`).
- **Wordlist**: Use the built‑in 200+ admin paths or import your own `.txt` file.
- **Match Filter**: Enter keywords or regex (e.g., `/root|admin/i`) to highlight successful hits and view context snippets.
- **Results Table**: Displays URL, match columns, size, status (color‑coded), and an "open" button. Rows are auto‑sorted (200 first, then errors last).

### 🛡️ NoRedirect Block‑list
Accessible via the `☰` button next to NoRedirect:
- **Simple match**: `admin.example.com` – blocks any URL containing this string.
- **Regex match**: `/^https?:\/\/[^\/]+\/admin\/?$/` – blocks matching URLs using regular expressions.
- Works even when the NoRedirect toggle is **OFF**.
